# SDD Security Review Log

Canonical tracking file for all SDD security reviews conducted by the Security team.

## How to Add an Entry

When a review is completed (or started), add a row to the table below:

1. **SDD Name** — short name for the design, matching the folder in `reviews/`
2. **Design Doc** — link to the design document (Notion, Confluence, Google Doc, etc.)
3. **Status** — `Reviewed`, `In Progress`, `Pending`, or `Not Reviewed`
4. **Risk Rating** — `High`, `Medium`, or `Low`
5. **Sec Relevant** — `Yes` or `—`
6. **Reviewer** — GitHub handle(s) of the Security team member(s)
7. **Review Date** — date the review was completed (or started, for in-progress)
8. **Ticket** — link to the issue tracking the review
9. **Review File** — relative path to the `review.md` or `notes.md` in this repo

> **Status key:**
> - `Reviewed` — security review completed
> - `In Progress` — review currently underway
> - `Pending` — queued for review
> - `Not Reviewed` — exists but has not been security reviewed
